Hello World!

This lesson is meant for absolute beginners in Python. We'll learn how to craft a function that says "Hello, World!".

Lesson objectives:

  • Create a .py file in the correct directory.
  • Understand how to define a function in Python.
  • Learn how to make a function return a value.

This lesson teaches you how to define a Python function. Try running the tests for this lesson before doing anything else!

Running Python Unit Tests

Here's a command that will allow you to run your tests from the top level directory of this project (the test_driven_python folder):

python -m unittest -vf tests.test_lesson_0_hello_world

That can look a little overwhelming at first, so let's break it down.

  • python indicates that we want to run something in Python
  • -m tells Python to run the module we want (test.test_lesson_x) as a script
  • unittest tells Python that we're running unit tests
  • -vf is a combination of two separate options:
    • -v encourages Python to be as verbose as possible with its output
    • -f tells Python to stop running the tests once it hits a failure. This is entirely optional, but you should try doing this to fix one failure at a time while you get the hang of how Python works
  • Finally, we tell Python which file it should run. In this case, that's tests.test_lesson_0_hello_world
What error did you get when you first ran that command?

Remember, errors are built to be helpful, not scary!

You should have seen an ImportError. This is Python telling you that it's unable to import the module hello_world because there is no hello_world.py file in the lesson 0 directory.

So your first step should be to create hello_world.py in lessons/lesson_0_hello_world.

Run the tests again. What's your next failure?

Defining a function in Python

Functions in Python are defined by the def keyword, followed by your function name. After your function name there are two parenthases followed by a colon, (): . If you're defining a function that accepts arguments this is where they'll go. Don't worry about arguments for now, though. We'll cover those in the next lesson.

Python operates using significant whitespace. This means that each time you enter a new function or logical block, you need to indent your code by 4 spaces. Python knows when you're done with your block when you unindent by 4 spaces.

def does_something():
    print "something"

Print vs Return

Two core concepts to think about when using Python are print and return.

print takes something in your function and prints it out into your terminal for you to read. This is a great way to see what's going on inide your function while it's running. print is incredibly useful for debugging, but does not provide your program with any useful information. print is for human use only.

return is the keyword that lets Python know that you're ready to exit the function with the value on that line. return provides useful information to the computer so that computed values can be passed from one function to another. Whenever you want to set something as the result of your function, use return. Remember that this will exit the function. You should never have any logic in your function that is unreachable after the return statement.

Note: if you've used other programming lanugages before, you might have come across the idea of implicit returns, meaning that the last variable in the top level scope of the function will always be returned. This is not the case in Python. If you want your function to return something, you must always explicitly state it, otherwise your function will return None.

Finishing the challenge

Now that you have your hello_world.py file, edit it to include a function that returns the string "Hello World!" Once you've edited your file, keep running your tests and amending your code until all of your tests pass. You'll know you're done when the output on your terminal reads:

Ran 2 tests in 0.000s OK

That means that you've passed all the tests and completed the challenge!
You're ready to move on to Lesson 1: Greetings!
